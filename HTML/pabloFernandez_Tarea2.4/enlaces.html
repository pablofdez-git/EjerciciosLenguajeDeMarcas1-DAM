Enlaces

URL (Uniform Resource Location)

Una URL es la dirección concreta de un recurso; página web, imagen, vídeo, etc, en Internet. Es un identificador absoluto y único para un recurso almacenable.

Sintaxis de una URL

La sintaxis general es: protocolo://usuario:contraseña@dominio:puerto/ruta?busqueda#fragmento.

  • Protocolo o esquema de red: Define el método para acceder al recurso.
  • Nombre de dominio: El nombre completo de la máquina o red que aloja el recurso.
  • Ruta: El recorrido dentro del servidor para acceder al archivo deseado.
  • Localizador interno (#fragmento): Un identificador que facilita el acceso a una parte específica dentro del recurso de destino.

Crear Enlaces en las Páginas Web con el Elemento <a>

El elemento <a> es esencial para crear hipervínculos. El atributo href define la URL de destino.

Tipos de Enlaces

  • Enlaces Globales o Externos: Referencian recursos ubicados en servidores diferentes al que aloja la página actual. (Ej: href="https://www.google.com").
  • Enlaces Locales Relativos o Internos: Referencian recursos ubicados en el mismo servidor. Toman como punto de partida el directorio de la página actual. (Ej: href="../pagina-secundaria.html").
  • Enlaces Internos al Propio Documento: Permiten saltar a un punto concreto dentro de la misma página. Requiere marcar la posición de destino con el atributo id. (Ej: href="#seccion-url").
  • Enlaces Internos a Otro Documento (Fragmentos): Combinan la URL de otro documento con un localizador interno. (Ej: href="otro-documento.html#seccion-clave").

Atributos Clave del Elemento <a>

  • href: Define la URL de destino.
  • target: Indica dónde abrir la URL enlazada:
    • _self: Carga en el mismo contexto de navegación (misma pestaña, es el valor por defecto).
    • _blank: Carga en un nuevo contexto de navegación (nueva pestaña).
  • download: Fuerza la **descarga** del recurso en lugar de abrirlo.
  • rel: Determina la relación entre el enlace y el recurso. Valores útiles: nofollow (para robots de búsqueda) y noreferrer.
