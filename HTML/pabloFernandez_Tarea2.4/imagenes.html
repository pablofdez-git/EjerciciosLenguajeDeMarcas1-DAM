Imagenes

El Elemento <img>

La etiqueta <img> es un elemento vacío (no tiene etiqueta de cierre) y se considera como texto al maquetarlo.

Atributos Principales

  • src: Es obligatorio e indica la URL (relativa o absoluta) a la imagen.
  • alt: Indica un texto alternativo que aparecerá cuando la imagen no se pueda cargar o mientras se carga. Es muy recomendable su uso, ya que ayuda a los buscadores y lectores de pantalla.
  • width y height: Permiten indicar la anchura y la altura de la imagen. No se recomienda su uso para redimensionar, ya que es mejor reducir la imagen previamente con un editor.
  • loading: Indica la estrategia de carga: eager carga inmediata, por defecto o lazy la carga se difiere hasta que la imagen está próxima a la visualización.
  • srcset y sizes: Permiten asignar varias imágenes junto con indicaciones para que el navegador elija la versión más adecuada para el dispositivo.

Imágenes Adaptativas: <picture> y <source>

El elemento <picture> es un contenedor que ofrece mayor versatilidad, facilitando la creación de imágenes adaptativas.

Contiene elementos <source> (para ofrecer diferentes opciones) y un elemento <img> final como respaldo.

El elemento <source> permite proporcionar versiones alternativas del contenido multimedia. Sus atributos clave incluyen:

  • type: Tipo MIM* del recurso. Si el tipo no se reconoce, el navegador no mostrará el recurso.
  • media: Permite determinar las reglas para seleccionar una u otra imagen en función de las características de la ventana o pantalla.

Mapas de Imagen: <map> y <area>

La técnica de mapas de imagen permite dividir una imagen en trozos para asignar un enlace a cada área. Hoy en día está en desuso.

  • Elemento <map>: Define el mapa de imagen. El atributo name es obligatorio y lo identifica.
  • Etiqueta <img>: Debe usar el atributo usemap con el nombre del mapa, anteponiendo el símbolo #.
  • Elemento <area>: Define cada región enlazable dentro del mapa.
    • shape: Forma del área: rect rectangular, circle circular o poly poligonal.
    • coords: Coordenadas del área, cuyo formato depende de shape.
    • href: URL de destino del enlace para esa área.

Elementos de Audio y Video: <video> y <audio>

HTML5 incorporó <video> y <audio>. El problema principal es la incompatibilidad de formatos entre navegadores.

Elemento <video>

Permite insertar videos. Puede contener elementos <source> para ofrecer diferentes fuentes de video y asegurar la compatibilidad.

  • controls: Si existe, proporciona una interfaz para que el usuario controle la reproducción.
  • autoplay: Indica que la reproducción debe comenzar automáticamente.
  • loop: Reproduce el video en bucle.
  • poster: URL de una imagen que se mostrará mientras el video no se está reproduciendo o se está descargando.
  • preload: Indica la precarga: auto, none o metadata.

Elemento <audio>

Permite insertar sonido. Sus atributos principales son: src, controls, autoplay, loop, y también puede contener etiquetas <source>.

Otras Formas de Incorporar Multimedia

<canvas>

Proporciona un lienzo sobre el que dibujar gráficos y animaciones mediante programas JavaScript.

  • Sus atributos clave son id, width y height.

<svg>

Permite incrustar gráficos vectoriales directamente en el documento HTML, definidos mediante etiquetas XML.

<iframe>

Permite incrustar contenido que se encuentra ubicado en un contexto distinto a la página que se está mostrando. Es muy popular en la actualidad, usado por ejemplo por YouTube.

  • src: URL al recurso incrustado.
  • allowfullscreen: Si existe, permite al contenido incrustado ocupar la pantalla completa.

<embed> y <object>

Se utilizan para colocar elementos no pertenecientes al lenguaje HTML (como animaciones Flash). <object> es la alternativa recomendada a <embed>.

El elemento <object> puede contener elementos <param> para especificar parámetros de presentación al objeto incrustado.
