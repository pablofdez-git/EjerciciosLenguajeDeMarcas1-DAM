Formularios

Elemento <form> y Funcionamiento

El elemento <form> es el contenedor que delimita los componentes interactivos que permiten al usuario introducir y enviar información. Cada control posee un nombre (atributo name) y un valor que se envía al servidor.

Atributos Clave del <form>

  • action: URL de la página o servicio al que se enviarán los datos para su procesamiento. Si los datos se manejan internamente con JavaScript, se puede usar action="#".
  • method: Determina el método de envío de los datos. Puede ser GET o POST.
    • GET: Los datos se pasan en la propia URL y son visibles en la barra de direcciones.
    • POST: Los datos se envían dentro del paquete HTTP y no por la URL.
  • autocomplete: Puede tomar los valores ON o OFF para que el navegador complete automáticamente los datos basándose en formularios anteriores.
  • novalidate: Si se usa, las validaciones automáticas del navegador no se llevarán a cabo.

Controles de Entrada: <input>

La mayoría de los controles de un formulario se insertan mediante el elemento <input>, cuyo tipo de control se define mediante el atributo type.

Los diferentes tipos de <input> ayudan al usuario en la introducción de datos y aportan seguridad mediante validaciones.

Atributos Comunes del <input>

  • name: Atributo fundamental que da un nombre al control. Si no se incluye, el contenido no será enviado.
  • id: Identificador único. Aconsejado para su uso con <label> y JavaScript.
  • required: Convierte el campo en obligatorio. Si no se rellena, la acción de envío se cancela.
  • placeholder: Muestra un texto informativo en el campo de entrada que desaparece cuando el usuario introduce un valor.
  • maxlength / minlength: Determinan la longitud máxima/mínima en campos de tipo texto.
  • pattern: Permite colocar una expresión regular para validar que el valor introducido cumple con un patrón específico.

Otros Controles de Formulario

Botones: <button>

Es la recomendación de HTML5. A diferencia de <input type="submit">, permite colocar código HTML en su texto.

  • type: Puede ser submit envío de datos, reset borrado de datos o button botón genérico.

Desplegables: <select> y <option>

Permiten seleccionar una o varias opciones de una lista.

  • <select>: Agrupa las opciones y da nombre al control. Atributos clave son multiple selección múltiple y required.
  • <option>: Define cada opción de la lista. El atributo value es el que se envía al servidor.
  • <optgroup>: Permite agrupar opciones dentro del desplegable.

Cuadro de Texto Multilínea: <textarea>

Permite colocar un cuadro de texto de varias líneas para la introducción de texto largo.

  • rows y cols: Indican la altura en líneas y la anchura en caracteres del cuadro de texto.

Agrupación y Etiquetado

  • <fieldset>: Permite agrupar un conjunto de controles en un formulario visual y funcionalmente, rodeándolos con un marco.
  • <legend>: Asigna un título o leyenda al marco de <fieldset>.
  • <label>: Asocia la etiqueta con un campo de entrada. La asociación se realiza usando el atributo for con el id del control, o conteniendo el campo de entrada.
  • <datalist>: Permite añadir entradas a un campo <input> para que actúen como sugerencia de escritura.

Controles Avanzados y Validaciones

Controles Avanzados

  • <output>: Se utiliza para mostrar resultados procedentes de cálculos sobre otros controles del formulario, típicamente con JavaScript.
  • <meter>: Muestra gráficamente un valor como fracción o parte de algo.
  • <progress>: Muestra gráficamente el grado de avance o cumplimiento de un proceso o tarea.

Validaciones

Las validaciones se realizan antes del envío del formulario (al pulsar un botón submit) para asegurar que los datos cumplen con las restricciones establecidas.

Las reglas se definen mediante atributos del control, como required, minlength/maxlength, min/max para números, type que valida formatos como email y pattern para expresiones regulares.
